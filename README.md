# 🚀 readmeGEN

**readmeGEN** — это интеллектуальный CLI-инструмент для автоматической генерации профессиональной технической документации прямо в терминале. Утилита сканирует структуру проекта, анализирует контекст исходного кода и генерирует адаптивный `README.md` с помощью моделей Llama через сверхбыстрое API Groq.

Инструмент ломает шаблонный подход стандартных ИИ-генераторов. Он не использует фиксированные структуры, а адаптирует разделы документации под архитектуру конкретного проекта: для CLI-утилиты на Ассемблере или C он сразу выведет флаги запуска, а для библиотеки на Python или JS — разберёт API и сигнатуры функций.

---

## 🎯 Чем он помогает разработчику?

* **⚡️ Экономия времени на рутине:** Никто не любит писать документацию вручную. Скрипт делает за вас 95% работы за 10 секунд.
* **🧠 Глубокий анализ кода (Context Awareness):** В отличие от обычных веб-чат-ботов, куда нужно копировать файлы руками, `readmeGEN` сам зарывается в проект. Он читает архитектурное дерево и верхушки исходных файлов, отсекая весь мусор вроде `__pycache__`, `node_modules` и виртуальных окружений.
* **🎭 Гибкая кастомизация на лету:** Инструмент поддерживает выбор языка (русский, английский, украинский), интерактивную настройку тона (от сухих фактов до подробного инженерного разбора) и позволяет задавать любые жесткие кастомные рамки (например, «сжать весь проект в один абзац»).
* **💻 Интерактивный терминальный UI:** Благодаря интеграции с библиотекой `rich`, вы получаете красивый рендеринг Markdown прямо в консоли с возможностью мгновенной отправки правок и фидбека нейросети без перезапуска скрипта.
* **🎨 Строгий визуальный кодекс:** Скрипт умеет точечно расставлять строгие технические эмодзи (`⚙️`, `📦`, `🔒`) исключительно для навигации, напрочь игнорируя раздражающий маркетинговый спам вроде `✨`, `🚀` и `🔥`.

## 🛠️ Установка и запуск

Для работы утилиты необходим установленный интерпретатор **Python 3.8+** и валидный API-ключ от платформы **Groq** (он бесплатный и выдается за пару кликов).

### 1. Клонирование репозитория

Склонируйте проект к себе на локальную машину:

```bash
git clone https://github.com/marpelskiy/readmeGEN
cd readmeGEN

```

### 2. Установка зависимостей

Для работы интерфейса и связи с моделью нужны всего две библиотеки. Установите их через стандартный пакетный менеджер:

```bash
pip install groq rich

```

### 3. Запуск утилиты

Просто запустите скрипт:

```bash
python readme_gen.py

```

При первом запуске скрипт вежливо попросит ввести ваш API-ключ Groq и надёжно сохранит его локально в вашей домашней директории (`~/.groq_readme_key`), чтобы вам не приходилось вводить его каждый раз или экспортировать переменные окружения.

## ⌨️ Как пользоваться

1. Запустите скрипт.
2. Укажите абсолютный или относительный путь к папке проекта, для которого нужно сделать документацию.
3. Выберите желаемый стиль (от сухого технического до кастомных ограничений).
4. Выберите язык и настройте предпочтения по эмодзи.
5. Изучите сгенерированный вариант прямо в терминале.
6. Если всё устраивает — выберите пункт `1` (Сохранить), и готовый `README.md` автоматически запишется в корень целевого проекта. Если нужно что-то поправить — выберите пункт `2` и в свободной форме напишите, что изменить.

## 🧩 Что если твоего языка нет в списке?

Даже если ты закинешь в скрипт проект на каком-то редком или специфическом языке, readmeGEN всё равно отработает отлично:

  1.  Модель получит полное дерево репозитория (project_tree), в котором отображаются абсолютно все файлы (кроме скрытых папок вроде .git). Нейросеть по именам файлов поймет структуру

  2. Модель определит лицензию проекта
